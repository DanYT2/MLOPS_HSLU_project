Predicting Customer Churn with MLOps

Customer churn costs companies billions of dollars annually. Customer acquisition costs 5-25x more than retention. This project builds an end-to-end Machine Learning pipeline to identify at-risk customers before they leave.

🛑

The Problem

Rule-based systems fail to capture complex, non-linear behaviors and are hard to scale.

🤖

The Solution

A production-grade ML pipeline using LightGBM + XGBoost ensemble to generate probabilistic churn scores.

⚙️

The Operations

MLflow tracking, Docker containers, and FastAPI serving ensure reproducibility and easy deployment.

The Data Landscape

I utilize the IBM Telco Customer Churn dataset (7,043 rows). Understanding the data structure and class imbalance is critical for selecting the right evaluation metrics (Recall & F1-Score) and preprocessing steps.

📂 Dataset Features

1. Demographic

Gender, SeniorCitizen, Partner, Dependents

2. Account Info

Tenure, Contract Type, PaperlessBilling, PaymentMethod, MonthlyCharges, TotalCharges

3. Services

PhoneService, InternetService, OnlineSecurity, TechSupport, StreamingTV, etc.

Class Imbalance Analysis

Target Variable: Churn (Yes/No)

Insight: Only ~26% of users churn. Accuracy is misleading here. We optimize for Recall.

Ensemble PyFunc Wrapper

Custom mlflow.pyfunc.PythonModel that loads all 10 fold models and averages predictions:

project/train.py — ChurnEnsembleModel
class ChurnEnsembleModel(mlflow.pyfunc.PythonModel):

    def load_context(self, context):
        self.lgb_models = [
            joblib.load(context.artifacts[f"lgbm_fold_{i}"])
            for i in range(1, 6)
        ]
        self.xgb_models = [
            joblib.load(context.artifacts[f"xgb_fold_{i}"])
            for i in range(1, 6)
        ]
        with open(context.artifacts["feature_list"]) as f:
            self.feature_list = json.load(f)

    def predict(self, context, model_input, params=None):
        df = model_input.reindex(
            columns=self.feature_list, fill_value=0
        )
        arr = df.values
        lgb_probs = np.mean(
            [m.predict_proba(arr)[:, 1]
             for m in self.lgb_models], axis=0
        )
        xgb_probs = np.mean(
            [m.predict_proba(arr)[:, 1]
             for m in self.xgb_models], axis=0
        )
        return (lgb_probs + xgb_probs) / 2

    name=REGISTERED_MODEL_NAME,
    alias="champion",

@asynccontextmanager
async def lifespan(app: FastAPI):
    tracking_uri = os.environ.get("MLFLOW_TRACKING_URI")
    if tracking_uri:
        mlflow.set_tracking_uri(tracking_uri)
    models_state["model"] = _load_registry_model(
        REGISTERED_MODEL_NAME, alias="champion"
    )
    yield
    models_state.clear()

app = FastAPI(
    title="Customer Churn Prediction API",
    description="LightGBM + XGBoost ensemble for churn prediction.",
    version="2.0.0",
    lifespan=lifespan,
)

@app.post("/predict", response_model=PredictionResponse)
def predict(customer: CustomerData):
    return _predict([customer])[0]

@app.post("/predict/batch", response_model=BatchPredictionResponse)
def predict_batch(customers: list[CustomerData]):
    if not customers:
        raise HTTPException(status_code=400, detail="Empty list")
    return BatchPredictionResponse(predictions=_predict(customers))

@app.get("/model/info")
def model_info():
    model = models_state.get("model")
    meta = model.metadata
    return {
        "registered_model": REGISTERED_MODEL_NAME,
        "model_uri": f"models:/{REGISTERED_MODEL_NAME}@champion",
        "run_id": getattr(meta, "run_id", None),
        "flavors": list(getattr(meta, "flavors", {}).keys()),
    }
from pydantic import BaseModel, Field
from typing import Literal

class YesNo(str, Enum):
    yes = "Yes"
    no = "No"

class InternetService(str, Enum):
    dsl = "DSL"
    fiber = "Fiber optic"
    no = "No"

class Contract(str, Enum):
    month = "Month-to-month"
    one_year = "One year"
    two_year = "Two year"

class CustomerData(BaseModel):
    SeniorCitizen: Literal[0, 1]
    Partner: YesNo
    Dependents: YesNo
    tenure: int = Field(ge=0)
    PhoneService: YesNo
    MultipleLines: MultipleLines
    InternetService: InternetService
    OnlineSecurity: InternetDependent
    OnlineBackup: InternetDependent
    DeviceProtection: InternetDependent
    TechSupport: InternetDependent
    StreamingTV: InternetDependent
    StreamingMovies: InternetDependent
    Contract: Contract
    PaperlessBilling: YesNo
    PaymentMethod: PaymentMethod
    MonthlyCharges: float = Field(ge=0)
    TotalCharges: float = Field(ge=0)

class PredictionResponse(BaseModel):
    churn_probability: float
    churn: bool

class BatchPredictionResponse(BaseModel):
    predictions: list[PredictionResponse]

Potential Development Challenges

Building an end-to-end MLOps pipeline for customer churn prediction is not only a technical task, but also an engineering and organisational challenge. Below are key risks that are likely to appear during development and deployment.

🧹

Data Quality & Drift

Inconsistent input formats, missing values, and delayed data feeds can break preprocessing pipelines. Over time, production data may drift away from the training distribution.

🏗️

Infrastructure Complexity

Integrating MLflow, Docker, FastAPI, and GitHub Actions requires careful environment management and consistent configuration across local and production.

⚖️

Metric Alignment

Business stakeholders may focus on accuracy, while the model is optimised for Recall and cost-based metrics. Aligning KPIs and explaining trade-offs is essential.

🔐

Security & Compliance

Handling customer-level data requires secure storage, controlled access to experiment artifacts, and compliance with governance and privacy regulations.

🧪

Testing & Reliability

Writing robust unit and integration tests for data pipelines, feature transformations, and API endpoints is non-trivial. Insufficient coverage can hide critical bugs.

🤝

Operational Adoption

Even with a strong model, integrating churn scores into existing CRM or retention workflows and closing the feedback loop can be challenging.
