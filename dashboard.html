Predicting Customer Churn with MLOps

Customer churn costs companies billions of dollars annually. Customer acquisition costs 5-25x more than retention. This project builds an end-to-end Machine Learning pipeline to identify at-risk customers before they leave.

🛑

The Problem

Rule-based systems ("tenure < 6 months") fail to capture complex, non-linear behaviors and are hard to scale.

🤖

The Solution

A production-grade ML pipeline using XGBoost to generate probabilistic churn scores (e.g., "85% Risk").

⚙️

The Operations

Automated training, testing, and deployment (CI/CD) ensures the model adapts to new data patterns.

The Data Landscape

I utilize the IBM Telco Customer Churn dataset (7,043 rows). Understanding the data structure and class imbalance is critical for selecting the right evaluation metrics (Recall & F1-Score) and preprocessing steps.

📂 Dataset Features

1. Demographic

Gender, SeniorCitizen, Partner, Dependents

2. Account Info

Tenure, Contract Type, PaperlessBilling, PaymentMethod, MonthlyCharges, TotalCharges

3. Services

PhoneService, InternetService, OnlineSecurity, TechSupport, StreamingTV, etc.

Class Imbalance Analysis

Target Variable: `Churn` (Yes/No)

⚠️ Insight: Only ~26% of users churn. Accuracy is a misleading metric here. We must optimize for Recall.

Potential Development Challenges

Building an end-to-end MLOps pipeline for customer churn prediction is not only a technical task, but also an engineering and organisational challenge. Below are key risks that are likely to appear during development and deployment.

🧹

Data Quality & Drift

Inconsistent input formats, missing values, and delayed data feeds can break preprocessing pipelines. Over time, production data may drift away from the training distribution, degrading model performance.

🏗️

MLOps Infrastructure Complexity

Integrating DVC, MLflow, Docker, FastAPI, and GitHub Actions requires careful environment management, secure secrets handling, and consistent configuration across local, CI, and production environments.

⚖️

Metric Alignment

Business stakeholders may initially focus on accuracy, while the model is optimised for Recall and cost-based metrics. Aligning KPIs and explaining trade-offs between False Positives and False Negatives is essential.

🔐

Security & Compliance

Handling customer-level data requires secure storage, controlled access to experiment artifacts, and compliance with internal governance and privacy regulations.

🧪

Testing & Reliability

Writing robust unit and integration tests for data pipelines, feature transformations, and API endpoints is non-trivial. Flaky tests or insufficient coverage can slow down CI/CD and hide critical bugs.

🤝

Operational Adoption

Even with a strong model, integrating churn scores into existing CRM or retention workflows, training teams, and closing the feedback loop between business and data science can be challenging.
